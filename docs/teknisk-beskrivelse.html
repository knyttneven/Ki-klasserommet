1. Prosjektoversikt

KI i klasserommet er et nettbasert kursverktøy for lærere i videregående skole. Formålet er å gjøre Microsoft Copilot praktisk og tilgjengelig gjennom prompt-kort, en faglig veiviser, en personlig prompt-bank og en veiledningsportal (Kompasset).

Løsningen er bygget som en statisk nettside uten byggprosess og kan hostes på GitHub Pages, intern webserver, SharePoint, eller ethvert filbasert webhosting-miljø. I versjon 3 er det lagt til en valgfri tilkobling til Supabase for delt innhold, anonym progresjonssporing og moderert brukerbidrag. En liten serverless backend håndterer sikker admin-sletting.

Alle Supabase-funksjoner er styrt av feature flags i config.js og er slått av som standard. Systemet fungerer fullt ut uten dem — statiske filer er alltid grunnlaget.

4. Sider og komponenter

4.1 index.html — Kurskartet

Hovelsiden. Viser et interaktivt øykart med 6 tematiske «øyer» (Kom i gang, Undervisning, Vurdering, Tilpasset opplæring, Koding, Kildebruk). Hver øy inneholder prompt-kort som lastes fra data.js.

  • Øyer rendres som SVG-elementer; klikk åpner en zoom-visning med kortliste
  • «Kildebruk»-øyen er skjult bak en tåke-mekanisme (fog-of-war) — aktivert som standard for alle vanlige brukere
  • Tåken løftes automatisk når brukeren har fullført CONFIG.fogUnlockAt kapitler, eller umiddelbart i admin-modus
  • Brukere kan valgfritt aktivere «Se hele kartet» — en lokal oversiktsmodus lagret i localStorage som ikke påvirker andre
  • Kartet er scrollbart og pannerbart; brukere trenger ikke bruke nettleser-zoom for å se innholdet
  • Fremdrift lagres i localStorage under nøkkelen ki-state; ved Supabase-aktivering også i kurs_progresjon via pseudo-ID
  • Hver bruker har individuell progresjon — fullføring på én enhet reflekteres ikke på en annen
  • Prompt-kort kan lagres til «Min bank» via modal
  • Inneholder admin-panel (PIN-beskyttet, se §7)
  • Laster: config.js → data.js → inline <script> + <script type="module">

4.2 veiviser.html — Veiviseren

Et stegbasert skjema der læreren velger fag, klassetrinn og pedagogisk mål. Genererer et skreddersydd prompt basert på valgene, med direkte lenke til riktig læreplan på udir.no.

  • All fagdata og kompetansemål hentes fra FAG-arrayen i data.js
  • Fag grupperes i <optgroup> basert på FAG_GRUPPER
  • Generert prompt kan kopieres med ett klikk og lagres i «Min bank»
  • Laster: config.js → data.js → inline <script>

4.3 min-bank.html — Min bank

Personlig prompt-bibliotek. Alle lagrede prompts vises som kort med navn, kilde og tidsstempel. Prompts kan slettes individuelt eller eksporteres som .txt-fil.

  • Data hentes fra localStorage, nøkkel ki-bank (JSON-array)
  • Kun klientsidelagring — ingen synkronisering mellom nettlesere
  • Laster: config.js → inline <script>

4.4 kompasset.html + kompasset.js — Kompasset

En veiledningsportal med kategoriserte tips, guider og ressurser for lærerne. Innholdet er strukturert i kategorier og tips-blokker med støtte for tekst, lister, kode-bokser, bilder og videoinnbygging.

  • HTML-skall i kompasset.html; all logikk og rendering i kompasset.js
  • Startdata fra kompasset-data.js; admin-endringer lagres som overlay i localStorage
  • Admin kan laste opp bilder (base64), importere/eksportere JSON, og publisere direkte til GitHub Pages via GitHub API
  • GitHub-token lagres i sessionStorage (slettes ved lukking av fane); repo-URL i localStorage
  • Laster: config.js → kompasset-data.js → kompasset.css → kompasset.js

6.2 Delt lagring via Supabase (valgfritt)

Når Supabase-funksjonalitet er aktivert i config.js, brukes følgende tabeller i tillegg til klientlagring:

TabellInnholdHvem kan skrive
kompass_tipsBrukerbidrag til KompassetAlle (anonymt)
kurskart_cardsBrukerbidrag til KurskartetAlle (anonymt)
kurs_progresjonØy-progresjon knyttet til pseudo-IDEier av pseudo-ID

Supabase bruker Row Level Security (RLS): lesing og skriving er tillatt uten innlogging; sletting krever serversiden admin-endepunkt med service_role-nøkkel.

Min bank er kun lagret lokalt i nettleseren og synkroniseres ikke mellom enheter. Eksporter .txt-filer for sikker langstidslagring. Øy-progresjon via Supabase er knyttet til denne nettleseren og dette tilfeldig genererte pseudo-IDet — sletting av nettleserdata fjerner koblingen.

7. Admin-panel og moderering

Admin-panelet er tilgjengelig på alle sider. Aktiveres ved å klikke på logoen 3 ganger raskt, eller via «Admin»-knappen (skjult by default).

Funksjonalitet

  • Kurskartet: Legg til egne prompt-kort; eksporter oppdatert JSON
  • Kompasset: Legg til/rediger/slett tips; last opp bilder; importer/eksporter JSON; publiser kompasset-data.js direkte til GitHub via API
  • Moderering (Supabase): Slett brukerbidrag fra Kompasset og Kurskartet via sikker sletting gjennom backend-endepunkt

PIN-kode

PIN settes i config.js under ADMIN_PIN. Standard er "2030" – endre dette før produksjonssetting.

Sikker sletting av Supabase-bidrag

Brukerbidrag som er lagret i Supabase kan ikke slettes direkte fra nettleseren — det blokkeres av Row Level Security. I stedet sendes en forespørsel til et eget backend-endepunkt (adminDeleteUrl i config.js). Admin oppgir et separat admin-passord som verifiseres på serveren. Bare ved korrekt passord utfører serveren slettingen på vegne av admin.

Dette betyr at Supabase-nøkkelen med fulle rettigheter aldri eksponeres i nettleseren.

Admin-PIN er ikke kryptering og gir ikke sikkerhet mot teknisk kyndige brukere som inspiserer kildekoden. Panelet er ment for å hindre utilsiktet redigering. Admin-passordet for Supabase-sletting er separat og lagres kun som miljøvariabel på serveren — aldri i kodebasen.

9. Deployment – GitHub Pages

9.1 Flyt for vanlig oppdatering

1
Rediger datafil lokalt
Åpne data.js eller kompasset-data.js i teksteditor. Gjør endringer. Lagre med UTF-8 encoding.
2
Test lokalt
Åpne index.html direkte i nettleser, eller kjør en lokal HTTP-server (python -m http.server).
3
Commit og push
git add data.js && git commit -m "Oppdater prompt-kort" && git push
4
GitHub Pages bygger automatisk
GitHub Pages deployer endringen innen 1–3 minutter. Ingen manuell byggprosess.

9.2 Kompasset-publisering via admin-panel

1
Admin åpner Kompasset og logger inn med PIN
2
Admin skriver inn GitHub PAT og repo-URL
Token lagres i sessionStorage. Repo-URL lagres i localStorage.
3
Klikk «Publiser til GitHub»
JavaScript gjør en PUT til GitHub API: https://api.github.com/repos/{repo}/contents/kompasset-data.js
4
GitHub Pages deployer automatisk
Siden er oppdatert for alle brukere etter 1–3 minutter.
GitHub PAT-en må ha contents:write-tilgang til repoet. En fine-grained token med tilgang til kun dette repoet anbefales.

10. Migrering til intern webserver

Siden det ikke finnes noen backend eller byggprosess, er migrering til intern server svært enkel.

Steg-for-steg

  1. Kopier alle filer fra repoet til webserveren (Apache, Nginx, IIS, SharePoint, m.m.)
  2. Sørg for at config.js finnes (ikke i .gitignore på produksjonsserveren)
  3. Verifiser at alle filer serveres med korrekt MIME-type (text/html, text/javascript, text/css)
  4. Test at alle fire sider lastes korrekt

Hva som må tilpasses ved intern server

FunksjonStatusTiltak
Visning av sider✓ Fungerer uten endringer–
localStorage / brukerdata✓ Fungerer uten endringer–
Admin-panel (legge til kort)✓ Fungerer uten endringer–
Kompasset-publisering via GitHub API⚠ Ikke aktueltFjern GitHub-delen fra admin-UI, eller tilby filnedlasting i stedet
Oppdatering av datafiler→ ManueltRediger data.js / kompasset-data.js og last opp til server via FTP/SCP/filbehandler
Anbefalt arbeidsflyt på intern server: Admin eksporterer JSON fra admin-panel → rediger lokalt ved behov → last opp oppdatert data.js / kompasset-data.js til serveren.

IIS-spesifikt (Windows Server)

  • Legg til MIME-type for .js: application/javascript (er vanligvis allerede satt)
  • Ingen spesielle rewrite-regler nødvendig (ingen routing, kun statiske filer)
  • Sett opp HTTPS-sertifikat – anbefales selv på intern server

11. Konfigurasjon

All konfigurasjon skjer i config.js via CONFIG-objektet:

export const CONFIG = {
  ADMIN_PIN:    "2030",          // PIN for admin-panel — endre før produksjon
  fogUnlockAt:  3,               // Øyer som må fullføres for å låse opp Kildebruk
  copilotUrl:   "https://copilot.microsoft.com",
  siteName:     "KI i klasserommet",
  siteSubtitle: "Praktiske prompt-kort for lærere i videregående",
  kontakt:      "",              // Valgfri e-post/URL for kontaktlenke

  adminDeleteUrl: "",            // URL til deployert backend/admin-delete.js

  supabase: {
    enableRead:        false,    // Les delt innhold fra Supabase
    enableWrite:       false,    // Skriv brukerforslag til Supabase
    enableProgression: false     // Lagre øy-progresjon i Supabase
  }
};

Supabase-flaggene er alle false som standard. Sett dem til true én etter én når du er klar til å aktivere funksjonaliteten. adminDeleteUrl settes til URL-en for det deployerte backend-endepunktet.

config.js er en ES-modul (export const CONFIG) og lastes med import i modul-scripts. Merk at config.js er committet til repoet — legg aldri sensitiv informasjon her utover admin-PIN. Backend-hemmeligheter (Supabase service_role-nøkkel, admin-passord) lagres kun som miljøvariabler på serveren.

12. Vedlikehold

Oppdatere prompt-kort (data.js)

  1. Åpne data.js i teksteditor
  2. Finn riktig øy i ISLANDS-arrayen
  3. Legg til eller rediger et kort-objekt (se mal i toppen av filen)
  4. Lagre med UTF-8, test lokalt, push til GitHub

Oppdatere fagkoder (udir.no)

  1. Finn fagkoden i FAG-arrayen i data.js
  2. Oppdater kode og udir-feltet
  3. Oppdater kompetansemaal-arrayen ved behov

Oppdatere Kompasset-innhold

  • Via admin-panel: Innlogg → rediger/legg til tips → eksporter JSON → last opp kompasset-data.js
  • Direkte i fil: Rediger kompasset-data.js, følg eksisterende blokkstruktur (se §5.3)

Legge til ny øy

  1. Legg til objekt i ISLANDS-arrayen i data.js
  2. Legg til tilhørende SVG-element i øykartet i index.html
  3. Oppdater <select id="av-island"> i admin-panelet
Viktig: Variabelnavnene i data.js (const FAG, const ISLANDS osv.) må ikke endres. Disse brukes direkte av JavaScript-logikken i HTML-filene.
