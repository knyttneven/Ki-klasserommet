KI i klasserommet

A practical AI prompt tool for upper secondary school teachers in Norway, built around Microsoft Copilot.

About

KI i klasserommet ("AI in the Classroom") is a static web application that helps Norwegian upper secondary school teachers use Microsoft Copilot more effectively in planning and teaching. It provides ready-made prompt cards organized by topic, a curriculum-aligned prompt wizard, a personal prompt library, and a guidance portal.

The application requires no backend, no build process, and no dependencies. It runs entirely in the browser and can be hosted on GitHub Pages, a school web server, SharePoint, or any static file host.

Features

  • Kurskartet — an interactive island map with 6 thematic chapters, each containing prompt cards for teachers
  • Veiviseren — a step-by-step wizard that generates a tailored Copilot prompt based on subject, year level, and pedagogical goal, with direct links to the Norwegian national curriculum (udir.no)
  • Min bank — a personal prompt library stored locally in the browser, with export to .txt
  • Kompasset — a guidance portal with categorised tips, how-to guides, and video resources for teachers
  • PIN-protected admin panel for adding prompt cards and publishing content updates
  • Progress tracking with a "fog of war" unlock mechanic
  • Accessible (ARIA roles, keyboard navigation, skip links)

Configuration

Copy config.example.js to config.js and edit the values:

const CONFIG = {
  ADMIN_PIN:    "your-pin",
  fogUnlockAt:  3,
  copilotUrl:   "https://copilot.microsoft.com",
  siteName:     "KI i klasserommet",
  siteSubtitle: "Praktiske prompt-kort for lærere i videregående",
  kontakt:      ""
};
Do not commit config.js with a real PIN to a public repository. config.js is listed in .gitignore by default.

Customising content

Prompt cards

Edit data.js. Each card in ISLANDS[n].cards follows this structure:

{
  id:     "und-01",
  title:  "Card title",
  time:   "5 min",
  out:    "What the teacher gets",
  do:     "Instruction for the teacher",
  prompt: "You are a teacher in [SUBJECT]…",
  tags:   ["Vg1", "Norwegian"],
  udir:   "NOR01-06"
}

Guidance content (Kompasset)

Edit kompasset-data.js, or use the in-app admin panel to add and edit tips without touching the source files. Tips support text, lists, code blocks, images (URL or base64), YouTube embeds, and links.

Subjects and curriculum links

Update the FAG array in data.js when Udir changes curriculum codes. Each entry holds the subject name, code, curriculum URL, and 2–4 sample competence goals.

Admin panel

Access by triple-clicking the site logo, or via the hidden Admin button. Requires the PIN set in config.js. Allows adding prompt cards, editing Kompasset content, and — when hosted on GitHub Pages — publishing kompasset-data.js directly to the repository via the GitHub API (requires a Personal Access Token with contents:write scope).

Data & privacy

All user data (progress, saved prompts, admin edits) is stored exclusively in the user's browser via localStorage. No data is sent to any server. There is no user tracking, no cookies, and no analytics.

Hosting on a school web server

Copy all files to the server root. No build step, no server-side code, and no special configuration required beyond standard static file serving. HTTPS is recommended. The GitHub publishing feature in Kompasset is not needed and can be ignored when self-hosting.
